# 新旧地图体系隔离

## 为什么必须隔离

工程目前同时存在两套路由：

- 老地图体系
  - 以 `blankCanvas` 为代表
  - 设施、规则、碰撞、渲染可以使用旧的简化实现
- 新地图体系
  - 以 `rmuc2026` 为代表
  - 设施必须从新地图的组合体、互动组件和注释读取

如果两套链路混在一起，最典型的问题就是：

- 新地图里出现老地图的粗模
- 新地图里还能锁到旧设施目标
- 新地图规则和新地图视觉彼此不一致

## 当前隔离原则

### 地图入口隔离

主界面只暴露需要的地图选项，不把旧实验地图的设施链路自动带入新地图。

### 设施来源隔离

在 `rmuc2026` 中，能量机关、前哨站、基地等设施必须来自：

- 新地图组合体
- 新地图互动组件
- 新地图注释配置

而不是从旧规则里硬编码生成一份替代物。

### 目标生成隔离

新地图中的结构目标应当由运行时组合体位姿导出，不能回退到旧结构目标生成器的固定几何。

### 碰撞隔离

新地图地形与组合体碰撞优先走新地形三角面/新组合体链路，不能再依赖旧的粗略方盒或老地图静态设施体积。

## 实际执行方式

- 地图加载阶段先识别地图类型
- 根据地图类型选择对应的缓存、注释、设施接入器
- 对局运行时按地图类型决定使用哪套渲染与规则设施来源

## 开发要求

以后如果新增地图，不要直接往旧逻辑里叠分支，而是先回答两个问题：

1. 新地图是否沿用旧设施体系
2. 如果不沿用，是否已经给出独立的组合体/互动组件/目标导出链路

只有这两个问题明确了，地图接入才不会再次互相污染。
