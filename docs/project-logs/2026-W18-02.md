# 2026-W18 研发记录 02

## 2026-04-27 局内 120FPS 目标性能优化

### 日志检查结论
- 检查目录：`build_verify/launcher_builds/debug/threeD/logs`。
- `frame_pump.log` 最近窗口显示局内平均间隔常落在 `12-18ms`，低于 120FPS 目标。
- `render_perf.log` 的主要尖峰来自 GPU 叠加层：常规 `overlay=14-31ms`，历史极端可到 `100ms+`；地形主渲染通常只有 `0.3-1.7ms`，单位渲染约 `1-4ms`。
- `simulation_perf.log` 后段仿真通常约 `1.6-2.7ms`，规则链路大多低于 `1ms`。
- `motion_control_slow.log` 中 AI 自瞄完整重扫出现 `8-11ms` 尖峰，主要集中在锁定目标后的重复瞄准链路。

### 调度优化
- `Simulator3dForm.cs`
  - 局内帧节拍不再用粗粒度 `Thread.Sleep` 追 120Hz，避免 Windows 计时粒度把 8ms 目标睡成 12-16ms。
  - 活跃对局改为短让出 / 轻自旋节拍；暂停、主界面和非局内流程仍保留睡眠，避免空转占满 CPU。
  - `frame_pump.log` 追加 `targetHz` 字段，方便直接核对运行目标。
  - 活跃局内 GPU 模式在 `Invalidate()` 后同步 `Update()` 提交绘制，减少 WinForms 消息队列合并造成的帧间隔抖动。

### 叠加层与资源管理
- `Simulator3dForm.GpuRenderer.cs`
  - GPU HUD 叠加层在活跃局内改为性能缩放表面，第一人称默认 `0.58x`，第三人称默认 `0.50x`；暂停、观察、战术、编辑和调试面板保持 `1.0x`，保证交互文字清晰。
  - 叠加层按逻辑尺寸和物理尺寸分开管理，窗口缩放或模式切换时才重建 bitmap/texture。
  - `render_perf.log` 增加 `overlay_detail(uploaded/scale/draw/tex/present/phases)`，把 GDI 绘制、纹理上传、呈现和慢阶段拆开看。
  - 交互态叠加层强制刷新，普通局内 HUD 继续按节流刷新，减少全屏位图上传。
  - 吊射副视角在扩展场景纹理路径下不再在 HUD 绘制阶段重复贴图，避免同一帧重复 OpenGL 拷贝。
- `Simulator3dForm.cs`
  - 局内活跃 GPU HUD 的左下状态面板不再每次重绘完整 3D 机器人小头像，改为低成本即时朝向头像，避免 `status` 阶段成为 50ms 级热点。

### 日志系统优化
- 新增 `src/Simulator.Core/SimulatorRuntimeLog.cs`。
- 运行日志从同步 `File.AppendAllText` 改成异步批量队列写入，覆盖：
  - `frame_pump.log`
  - `render_perf.log`

  - 以及 `AppendGameplayLog(...)` 统一入口。
- 队列设置上限，避免异常日志洪峰反向拖垮主线程。

### AI 自瞄调度优化
- `TerrainMotionService.cs`
  - AI 锁定装甲板后，短时间内优先复用 EKF/残留预测解，约 `0.22s` 后再做完整候选和视线重扫。
  - 保留之前新增的移动目标跟踪感，同时减少 `motion_control_slow.log` 中重复 `aim=8-11ms` 的尖峰概率。
