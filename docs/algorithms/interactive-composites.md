# 组合体控制与互动组件运行时

## 什么是组合体

组合体是地图里的可动对象容器，例如：

- 能量机关
- 前哨站
- 基地顶部组件

组合体负责：

- 位置
- 旋转
- 坐标系
- 锚点

互动组件负责：

- 装甲板受击
- 圆盘环数反馈
- 灯条颜色
- 可视锁定目标

## 运行时原则

组合体链路必须遵守两个原则：

1. 组合体先于互动组件更新
2. 互动组件世界位姿由组合体矩阵推导，不允许独立漂移

也就是说，互动组件没有自己的“绝对真位置”，它的世界位置来自：

- 组件本地坐标
- 组合体当前位姿矩阵

## 典型流程

以能量机关为例：

1. 读取组合体主体与全部互动组件
2. 建立组合体本地坐标系
3. 每帧更新组合体位姿
4. 用组合体矩阵把圆盘、灯条变换到世界空间
5. 生成运行时目标与渲染反馈
6. 自瞄、命中、视觉反馈共同消费这些结果

前哨站与基地同理，只是运动规则不同：

- 前哨站主要是旋转
- 基地顶部组件主要是平移或开闭

## 组合体编辑

编辑器侧需要支持：

- 选择组合体
- 修改锚点
- 修改旋转中心
- 选择使用世界系还是自定义坐标系
- 将结果保存回本地配置

局内 `F9` 编辑则是运行时版的轻量入口，目的是快速校正组合体位置并落盘。

## 为什么一定要统一矩阵

只要下面任意一项使用了另一套矩阵，就会出问题：

- 渲染
- 命中
- 自瞄
- 编辑器保存

表现通常是：

- 锁定点漂移
- 命中反馈不在模型上
- 组合体本体在一个地方，装甲板在另一个地方

因此当前实现里，运行时组合体位姿矩阵已经被提升为整个设施链路的核心中间结果。

## 2026-04-25：实体初始化矩阵统一

组合体和互动组件不应只在渲染层拥有矩阵。当前 `SimulationWorldState` 会记录全局 `model_to_scene_metric` 与 `model_to_world_units`，并在实体初始化时同步到每个机器人和设施实体。

这样做的目的有三个：

- 后续新建机器人或设施时默认继承同一套坐标变换。
- 互动组件、碰撞体、自瞄目标和 HUD 虚影可以追溯到同一个矩阵来源。
- 如果之后再次出现整体偏移，可以先检查世界状态中的矩阵，而不是分别排查每个组合体的经验位移。

需要注意的是：本轮只是把矩阵作为统一事实记录下来，不额外移动视觉模型。视觉模型位置仍由地图编辑器导出的模型坐标和渲染缓存决定；如果碰撞体偏移，只在碰撞层修正。

## 前哨站死亡停转

前哨站的旋转角现在由规则层统一解算：

- 存活时按固定角速度转动
- 死亡后按阻尼曲线逐渐停转

渲染层与自瞄层共同使用这条曲线，这样不会再出现：

- 模型已经减速
- 自瞄还按恒速追板
