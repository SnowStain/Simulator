# 弹丸与模型碰撞

## 目标

弹丸系统需要同时满足三件事：

- 够快：能在高帧率下处理大量弹丸
- 够准：命中互动组件、装甲板、地形时结果一致
- 不分叉：渲染看到的模型与命中判定尽量共用同一套空间结果

## 基本流程

每帧弹丸更新分为三步：

1. 位置积分
2. 沿运动方向做线段检测
3. 命中后再根据命中的实体类型决定伤害、反弹或交互反馈

也就是说，弹丸不会先对所有实体做完整昂贵求交，而是先沿当前速度方向做窄相位检测。

## 命中对象

当前支持的命中对象主要包括：

- 机器人装甲板
- 前哨站/基地结构装甲板
- 能量机关圆盘与互动组件
- 新地图三角形地形
- 其他设施组合体

## 地形碰撞

新地图的地形命中不再依赖粗栅格近似，而是优先使用精细地形数据导出的三角面。

核心思路是：

- 先通过空间索引筛掉大量不相关三角形
- 对弹丸运动线段与候选三角面做求交
- 记录法线、碰撞点和碰撞类型

这些结果既可以用于伤害判定，也可以用于后续的反弹或阻挡逻辑。

## 与组合体的关系

对于能量机关、前哨站、基地这类可动组合体，命中判定不能只依赖初始模型。

必须先拿到组合体当前帧的位姿变换矩阵，再把互动组件从模型空间变换到世界空间，最后再做命中判断。

否则就会出现：

- 肉眼看到命中
- 规则层判定没命中

或者反过来的情况。

## 小弹丸优化

小弹丸数量大，处理重点是减少无效开销：

- 复用数组，减少分配
- 合并绘制，减少 draw call
- 只对运动方向前方的候选物体做窄相位检测
